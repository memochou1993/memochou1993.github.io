гҖҢKubernetes еҜҰдҪңжүӢеҶҠпјҡеҹәзӨҺе…Ҙй–ҖзҜҮгҖҚStudy Notes
Deployment, Docker, Kubernetes, kubeadm

еүҚиЁҖ

жң¬ж–ҮзӮәгҖҢKubernetes еҜҰдҪңжүӢеҶҠпјҡеҹәзӨҺе…Ҙй–ҖзҜҮгҖҚиӘІзЁӢзҡ„еӯёзҝ’зӯҶиЁҳгҖӮ

е®үиЈқ kubeadm

жҢҮд»Ө kubeadm йҒ©еҗҲдёҖиҲ¬дҪҝз”ЁиҖ…еүөе»ә Kubernetes еҸўйӣҶпјҢдёҰдё”ж”ҜжҸҙе–®дёҖзҜҖй»һжҲ–еӨҡзҜҖй»һгҖӮ

ж №ж“ҡе®ҳж–№ж–Үд»¶пјҢйңҖиҰҒеңЁжҜҸеҸ°ж©ҹеҷЁдёҠе®үиЈқд»ҘдёӢеҘ—д»¶пјҡ

  • kubeadmпјҡз”ЁдҫҶеҲқе§ӢеҢ–еҸўйӣҶзҡ„жҢҮд»ӨгҖӮ
  • kubeletпјҡеңЁеҸўйӣҶдёӯзҡ„жҜҸеҖӢзҜҖй»һдёҠз”ЁдҫҶе•ҹеӢ• Pod е’Ңе®№еҷЁзӯүгҖӮ
  • kubectlпјҡз”ЁдҫҶиҲҮеҸўйӣҶйҖҡиЁҠзҡ„е‘Ҫд»ӨеҲ—е·Ҙе…·гҖӮ

йҰ–е…ҲпјҢжҢҮе®ҡиҰҒе®үиЈқзҡ„ Kubernetes зүҲжң¬гҖӮ

жӣҙж–° apt еҘ—д»¶зҙўеј•пјҢдёҰе®үиЈқдҪҝз”Ё Kubernetes apt еҖүеә«жүҖйңҖиҰҒзҡ„еҘ—д»¶пјҡ

дёӢијү Google Cloud е…¬й–Ӣз°ҪеҗҚеҜҶй‘°пјҡ

ж·»еҠ  Kubernetes apt еҖүеә«пјҡ

жӣҙж–° apt еҘ—д»¶зҙўеј•пјҢе®үиЈқжҢҮе®ҡзүҲжң¬зҡ„ kubeletгҖҒkubeadm е’Ң kubectlпјҡ

kubelet зҸҫеңЁжҜҸйҡ”е№ҫз§’е°ұжңғйҮҚе•ҹпјҢеӣ зӮәе®ғйҷ·е…ҘдәҶдёҖеҖӢзӯүеҫ… kubeadm зҡ„з„ЎзӘ®иҝҙеңҲгҖӮ

е•ҹеӢ•еҸўйӣҶ

жіЁж„Ҹ Pod з¶Іи·ҜдёҚиЎҢе’Ңд»»дҪ•дё»ж©ҹз¶Іи·ҜйҮҚз–ҠпјҢеҰӮжһңжңүиЎқзӘҒпјҢеңЁеҹ·иЎҢ kubeadm init жҢҮд»ӨжҷӮйңҖиҰҒдҪҝз”Ё --pod-network-cidr еҸғж•ёгҖӮ

kubeadm жңғй–Ӣе§ӢеңЁ /etc/kubernetes/manifests иіҮж–ҷеӨҫеүөе»әдёҖдәӣжӘ”жЎҲпјҢйҖҷдәӣ YAML жӘ”еҗ„иҮӘжҸҸиҝ°дәҶеӣӣеҖӢж ёеҝғе…ғд»¶пјҡеҢ…жӢ¬ API ServerгҖҒControllerгҖҒScheduler е’Ң etcdгҖӮ

kubelet жңғеҺ»и®ҖеҸ–йҖҷдәӣжӘ”жЎҲпјҢз„¶еҫҢйҖҸйҒҺ Docker еүөйҖ еҮәйҖҷдәӣ containerпјҢйҖҷдәӣ container дә’зӣёжәқйҖҡе°ұеҪўжҲҗдәҶ Kubernetes еҸўйӣҶпјҢжңҖеҫҢ kubeadm жңғйҖҸйҒҺе’Ң API Server жәқйҖҡзҡ„ж–№ејҸпјҢжҠҠ kube-proxy д»ҘеҸҠ DNS йҖҷе…©еҖӢ container йғЁзҪІеҲ° Kubernetes еҸўйӣҶд№ӢдёӯгҖӮ

зӮәдәҶи®“дёҖиҲ¬дҪҝз”ЁиҖ…д№ҹиғҪеӨ дҪҝз”ЁеҸўйӣҶпјҢеҹ·иЎҢд»ҘдёӢжҢҮд»ӨгҖӮ

жҺҘи‘—пјҢдҪҝз”Ёд»ҘдёӢжҢҮд»ӨеҸ–еҫ— Pod еҲ—иЎЁпјҢз”ұж–ј Kubernetes зҡ„ 4 еҖӢж ёеҝғе…ғд»¶жҳҜиў«ж”ҫеҲ°й җиЁӯзҡ„ namespaceпјҢеӣ жӯӨйңҖиҰҒжҢҮе®ҡ namespace зӮә kube-systemгҖӮ

жңҖеҫҢпјҢе•ҹеӢ•дёҖеҖӢ CNI еӨ–жҺӣпјҢжӯӨиҷ•дҪҝз”Ё Flannel еӨ–жҺӣгҖӮ

еҶҚжӘўжҹҘдёҖж¬Ў Pod еҲ—иЎЁпјҢжүҖжңүзҡ„ Pod зӢҖж…ӢйғҪе·Іжӣҙж–°зӮәжӯЈеңЁйҒӢиЎҢгҖӮ
